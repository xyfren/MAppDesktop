# MAppDesktop

Проект состоит из **двух частей**:
- **MAppServer** — серверное приложение, создающее виртуальный (второй) монитор.
- **MAppDriver** — драйвер виртуального дисплея, с которым сервер взаимодействует через **ioctl**.

Сервер используется как часть связки с проектом **MApp** — он формирует второй монитор и передает/принимает данные от клиента.

## Состав репозитория

- `MAppServer/` — серверное приложение (C++/WinAPI).
- `MAppDriver/` — драйвер виртуального дисплея (WDDM/IddSample).
- `MAppDesktop.sln` — решение Visual Studio.

## Требования

- Windows 10/11
- Visual Studio (рекомендуется 2019/2022)
- Права администратора для установки драйвера

## Сборка

1. Откройте `MAppDesktop.sln` в Visual Studio.
2. Соберите проекты `MAppDriver` и `MAppServer`.
3. Убедитесь, что драйвер собран (обычно `*.sys`, `*.inf`, `*.cat`).

## Установка драйвера

В репозитории (и в релизах) будет размещён `.bat` файл для установки драйвера.

**Установка:**
1. Запустите `.bat` файл **от имени администратора**.
2. Дождитесь установки драйвера.
3. Перезагрузите ПК при необходимости.

> Примечание: если Windows блокирует установку из‑за подписи, включите тестовый режим (см. ниже).

## Включение тестового режима Windows (Test Mode)

Это необходимо для установки неподписанных драйверов в тестовой среде.

Откройте **Командную строку от имени администратора** и выполните:

```
bcdedit /set testsigning on
```

Перезагрузите ПК.

После перезагрузки в правом нижнем углу появится надпись **Test Mode**.

**Отключить тестовый режим:**
```
bcdedit /set testsigning off
```
и снова перезагрузить ПК.

## Запуск сервера

1. Убедитесь, что драйвер установлен.
2. Запустите `MAppServer` (собранный `.exe`).
3. Сервер создаст виртуальный монитор и будет готов к работе с MApp.

## Связь с MApp

Сервер предназначен для работы в связке с проектом **MApp** и формирует второй монитор на ПК, с которым взаимодействует клиент.
